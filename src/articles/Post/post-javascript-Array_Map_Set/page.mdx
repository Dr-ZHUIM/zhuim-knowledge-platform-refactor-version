## Array、Map、Set 的性能比较

JavaScript 中的 Array、Map、Set 的性能表现依赖于不同的使用场景以及使用操作。

### Array 数组

- 访问： Array 通过索引访问元素，时间复杂度为 <Im>O(1)</Im>
- 插入/删除：
  - 在数组末尾插入或者删除相对较快，平均时间复杂度为 <Im>O(1)</Im>
  - 在数组开头或者中间插入删除则需要移动其余元素，平均时间复杂度为 <Im>O(n)</Im>
- 搜索： 通过线性查找数组中的元素，时间复杂度为 <Im>O(n)</Im>

### Map 映射

- 插入/删除/搜索： 映射提供高效的插入、删除和搜索操作。这些操作的时间复杂度通常为平均 <Im>O(1)</Im>，但在某些具有大量元素的情况下，时间复杂度可能增加到 <Im>O(log n)</Im>。

### Set 集合

- 插入/删除/搜索： 这些操作的时间复杂度通常为平均 <Im>O(1)</Im> 。然而，在某些情况下，当处理大量元素时，时间复杂度可能增加到 <Im>O(log n)</Im> ，其中 n 是集合中的元素数量。

### Map、Set 操作性能较高的原因

Map 、Set 在实现上使用红黑树，Map 和 Set 存储的所有元素都是以节点的方式来进行存储的，这种节点结构和链表有点类似。

两个节点通过指针相连，删除或者增加元素时，我们只是重新更改了指针的指向，不像数组那样，掺入或删除之后需要重新排序。
